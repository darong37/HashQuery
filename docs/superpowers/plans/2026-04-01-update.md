```perl
    if ($upd) {
        my %upd_cols = %{ $upd->{update} };

        my %valid = map { $_ => 1 } @all;
        for my $col (keys %upd_cols) {
            die "unknown column in UPDATE: $col"
                unless $valid{$col};
        }

        my $matched = $tbl;
        $matched = _run_where($matched, $as, $whr) if $whr;
        $matched = _run_having($matched, $as, $hvg) if $hvg;

        my %upd_idx = map { $_->{_idx} => 1 } @$matched;
        for my $row (@$tbl) {
            next unless $upd_idx{ $row->{_idx} };
            for my $col (keys %upd_cols) {
                $row->{$col} = $upd_cols{$col};
            }
        }

        my $result = _run_select($tbl, \@all);

        if ($as) {
            ${ $as->{alias} } = {
                count  => scalar @$result,
                affect => scalar @$matched,
            };
        }
        return $result;
    }
```

**セクション 3（各関数リファレンス）** に `### UPDATE` を追加（`### DELETE` の直後）:

```markdown
### UPDATE

**シグネチャ:**

```perl
UPDATE { col => val, ... };
```

**戻り値:**

```perl
{ update => { col => val, ... } }
```

**動作説明:**

`query` に渡すことで更新モードを宣言する。`where` / `having` にマッチした行の指定カラムを固定値で上書きし、全行（更新済み行・未更新行を含む）を AOH で返す。元のテーブルは変更しない。`SELECT` / `DELETE` と同時に指定した場合は die する。存在しないカラム名を指定した場合は die する。

**コード例:**

```perl
# SELECT で更新対象を確認
my $preview = query $table, SELECT, where { $_->{score} < 60 };

# UPDATE に差し替えて実行
my $result = query $table, UPDATE { score => 60, grade => 'C' }, where { $_->{score} < 60 };
```
```

**セクション 3（as）** の `| 操作 | count | affect |` テーブルに行を追加:
```
| `UPDATE` | 全行数 | 更新行数 |
```

**セクション 4（エラーハンドリング）** を更新:

旧:
```
| `select` と `delete` を同時に指定 | `"select and delete cannot be used together"` |
```
新:
```
| `SELECT` / `DELETE` / `UPDATE` を2つ以上同時指定 | `"SELECT, DELETE, and UPDATE cannot be used together"` |
| `UPDATE` に存在しないカラム名を指定 | `"unknown column in UPDATE: <カラム名>"` |
| `UPDATE` の引数がハッシュリファレンスでない | `"UPDATE requires a hash reference"` |
```

**セクション 7（実行モデル）** に UPDATE モードを追加:

```markdown
### UPDATE モード

1. 各行への `_idx` 付加（0 始まりの行番号）
2. `UPDATE` ハッシュのカラム名が全て既存カラムであることを検証（不正なら die）
3. `where` 評価（更新候補を特定）
4. `having` 前計算 → 評価（更新候補をさらに絞り込み）
5. 更新候補の各行に対して `UPDATE` ハッシュの値を代入
6. 全行（更新済み行・未更新行を含む）を AOH として返す（`_idx` を除外）
7. `as` 変数に `{ count => 全行数, affect => 更新行数 }` を格納
```

**セクション 8（制約・注意事項）** に追加:
```
- `SELECT`・`DELETE`・`UPDATE` は排他であり、2つ以上の同時指定は die する
- `UPDATE` モードでは列射影を行わない（全列を返す）
- `UPDATE` モードでも元テーブルは変更されない（非破壊的操作）
- `UPDATE` に存在しないカラムを指定した場合は die する
- `UPDATE` は固定値のみサポート。動的な計算は `query` 呼び出し前に Perl 側で行う
```
