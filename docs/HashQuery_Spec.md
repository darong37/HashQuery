# HashQuery Spec

## 0. 最重要前提（絶対条件）

- 実行可能なのは **`query` のみ**
- `select` / `as` / `where` / `having` は実行関数ではない
- これらは **`query` に渡すための DSLノード値** を表す
- `select ...` / `as ...` / `where { ... }` / `having { ... }` のような
  **DSLノード生成関数とその引数を合わせたもの** を **DSL部品** と呼ぶ
- DSL部品の戻り値は、すべて **深さ1のハッシュリファレンス** で統一する
- DSL部品の戻り値は、必ず自分自身のキーで構成する
- 各DSL部品は、**括弧なし** で記述することを最重要前提とする
- `select(...)` のような括弧付き記法を前提に設計しない
- `select` / `as` / `where` / `having` は、
  プロトタイプにより括弧なしで表現できることを前提にする
- `select` の引数は、配列そのものではなく **配列リファレンス** で渡す
- 実行は必ずこの形のみ

```perl
query $table, DSL部品...
```

- 外部エンジン・別実行層は一切存在しない

## 1. 提供関数

- `query`   `# 唯一の実行関数`
- `as`
- `select`
- `where`
- `having`

## 2. 基本原則

- 入出力はすべて **AOH（Array of Hash）**
- `query` の第一引数には、必ず **Table** を指定する
- ここでいう **Table** とは、AOH をデータテーブルとして扱うときの呼称である
- Table の全行は同一のカラム構成を持つ必要がある（列名・列数が一致していない場合はエラー）
- 第一引数の後には、各DSL部品を並べて指定する
- 行数制御は `where` / `having` のみ
- `select` は列制御のみ（行数は変えない）
- `from` は廃止する

```perl
my $table  = ...;
my $tbl;
my $result = query $table, DSL部品...;
my $cnt    = scalar @$result;
```

## 3. DSL構文ルール

- Perlプロトタイプで成立させる
- 各DSL部品は括弧なしで書けることを前提とする

```perl
sub query ($@);
sub as (\$);
sub select (;$);
sub where (&);
sub having (&);
```

- DSL部品は即時評価しない
- 実行は `query` のみが行う

## 4. 基本構文

```perl
query
    $table,
    as $tbl,
    select [qw/a b c/],
    where  { ... },
    having { ... };
```

- `as` は Table に対する指定なので、Table の直後に置く
- 実行順序は表示順とは別である

## 5. 実行順序

1. `as`
2. `select`（`except` を含む）の解釈と出力カラム確定
3. `where`
4. `having`（前計算 → 評価）
5. `select`（列射影）

- DSL記述順には依存しない

## 6. query

```perl
query $table,
    DSL部品...;
```

- 第一引数
  - Table（AOH）

- 第二引数以降
  - 各種DSL部品（深さ1のハッシュリファレンス）

- 戻り値
  - AOH

- 役割
  - 第一引数の Table を対象に、後続の各種DSL部品を解釈して実行する

- `_row` カラムについて
  - `query` は実行時、Table の各行に内部カラム `_row`（0始まりの行番号）を自動付加する
  - `_row` は `where` / `having` の評価中に `$tbl->{_row}` として参照可能
  - `_row` は `grep_concat` が相対行を取得するための基点として使用する
  - 最終出力（AOH）には含まれない

## 7. as

```perl
my $tbl;
as $tbl;
```

- 引数
  - スカラー変数への参照（\$var 形式）

- 戻り値

```perl
{
  alias => \$var  # スカラー変数への参照（SCALAR REF）
}
```

- 役割
  - `where` / `having` 内で利用する Table のエイリアス変数を指定する

- 仕様
  - `as` は単なるノード値であり、実行しない
  - エイリアスという考え方は従来のものを踏襲する
  - `as` は、文字列ではなく変数参照として受け取る
  - `as $tbl` と指定した場合、`where` / `having` の中では `$tbl` によって現在行を参照できる
  - `where` / `having` において主役になるのは、`as` で指定したエイリアス変数である
  - `as` に渡した変数には、現在行の hash リファレンスが入るものとして扱う
  - `as` による alias 変数は、`where` と `having` の両方で同じ考え方で使えるようにする

## 8. select

```perl
select [qw/a b c/];
```

- 引数
  - 配列リファレンス（カラム名の明示指定）
  - `except` キーを持つハッシュリファレンス（除外指定）
  - スカラー文字列 `'*'`（全列指定）
  - 引数なし（全列指定、`'*'` と同義）

- 戻り値

```perl
{
  select => [qw/a b c/]
}
```

- 役割
  - 出力に含めるカラムを指定する

- 仕様
  - `select` は単なるノード値であり、実行しない
  - 行数は変えない
  - `select` の受け方は次の3種類とする
    - 配列リファレンス
    - `except` を持つハッシュリファレンス
    - スカラー `'*'`
  - スカラーで受ける場合は `'*'` のみ許可する
  - `query` は初期段階で、第一引数の Table を見て出力カラムを確定する
  - `select` / `except` のどちらで指定された場合も、出力カラムの最終確定は `query` が行う

- 明示指定

```perl
select [qw/a b c/];
```

- 戻り値

```perl
{
  select => [qw/a b c/]
}
```

- `except` 指定

```perl
select {
    except => [ 'c' ]
};
```

- 戻り値

```perl
{
  except => [ 'c' ]
}
```

- 仕様
  - `except` は `select` の中でのみ使う
  - `except` が指定された場合は、`query` 実行時に Table の全列からその列を除外した結果を出力カラムとして確定する
  - 明示カラム指定は、配列リファレンスで渡す
  - `select` / `except` は、`query` から Table を見える前提で実装する
  - `select` のために mode や node 単位の追跡情報を別途持つことを前提にしない
  - `query` は `select` または `except` を受け取った時点で出力カラムを確定し、その確定結果を後続処理でも利用できるようにする

- 全列

```perl
select;
select '*';
```

- 仕様
  - `select` または `select '*'` は全列指定を表す
  - `select` に何も指定しない場合と `select '*'` は同義とする
  - 実際の全列の確定は query の初期段階で Table を基準に行う

## 9. where

```perl
where {
    $tbl->{a} > 10
};
```

- 引数
  - コードブロック（真偽値を返す条件式）

- 戻り値

```perl
{
  where => CODE
}
```

- 仕様
  - Table を1行ずつ評価する
  - trueのみ残す
  - `as` が指定されている場合は、その alias に対応する変数が主役になる
  - `as` がある場合は alias を優先する。ただし `$_` にも同じ view が設定されるため `$_` 経由での参照も可能
  - 例: `as $tbl` が指定されている場合、`where` の中で `$tbl->{a}` と書ける
  - `as` を指定しない場合は `$_` によって現在行を参照する
  - `$tbl` や `$_` は、plain な hash リファレンスではなく `HashQuery::RowHash` のインスタンスとして扱う
  - `$tbl->{a}` や `$_->{a}` が返す値は `HashQuery::Value` のインスタンスであり、`like` / `between` / `in` などを呼べる値オブジェクトである
  - `where` の実装では、`as` で受けた変数参照と `$_` の両方に `HashQuery::RowHash` インスタンスを割り当てる
  - `where` のために caller package や node package のような追跡機構を前提にしない

- whereで使用できる関数

```perl
$tbl->{name}->like('ab%')
$tbl->{score}->between(10, 20)
$tbl->{kind}->in([ 'x', 'y' ])
$tbl->{name}->asNull('unknown')
$tbl->grep_concat('error', 0, 1)
$tbl->{kind}->not_in([ 'x', 'y' ])
```

※ 上記は `as $tbl` を指定した場合の記述例である

- 仕様
  - `where` では、現在レコードを基準に評価される関数を利用できる
  - `where` で使用できる関数は、真偽値を返す条件関数に限定しない
  - `where` で使用できる関数は、少なくとも次の系統に整理する
    - 値条件関数
    - 値変換・加工関数
    - レコード単位のコンテキスト関数
  - 現時点で使用可能な値条件関数は次のとおり
    - `like`
    - `between`
    - `in`
    - `not_in`
    - `not_like`
  - 現時点で使用可能な値変換・加工関数は次のとおり
    - `asNull`
  - 今後追加する新しい関数は、現在レコードを起点に評価・参照・加工を行うものであれば、`where` 句内で使用可能な関数群として同じレイヤーに配置してよい
  - 新しい関数は、既存の `like` / `between` / `in` と同じ真偽値関数カテゴリに揃える必要はない
  - 新しい関数が文字列取得や周辺参照を行う場合でも、判定主体が現在レコードである限り `where` 相当の関数として扱う
  - 前後行参照のように現在レコードの周辺コンテキストを参照する関数は、`where` 句内のレコード単位コンテキスト関数として整理する
  - 拡張予定の関数として、少なくとも次を想定する
    - `is_empty`
    - `is_blank`
  - `is_` が付く関数は、真偽値を返す関数として扱う
  - `asNull` は真偽値関数ではなく、対象が Null だった場合の出力を扱う別種の関数として区別する
  - `between` は境界値の排他指定をサポートする
    - `A, B` は通常の範囲指定（下限以上かつ上限以下）
    - `A!, B!` のように値の後ろへ `!` を付けると、その境界値を排他（含まない）として扱う
    - この `!` は演算子ではなく、文字列の末尾に付加する記法として扱う
  - これらは SQL の演算子に対応する条件指定として扱う

## 10. 値アクセス機構

`where` / `having` 内で `$tbl` や `$_` と参照するオブジェクトは `HashQuery::RowHash` のインスタンスである。`$tbl->{col}` のようにカラムを添字で参照すると `HashQuery::Value` のインスタンスが返る。

### HashQuery::RowHash

- 役割
  - `$tbl` および `$_` の実体
  - `tie` によるハッシュアクセスを提供する
  - カラムを添字で参照すると `HashQuery::Value` を返す
  - `grep_concat` メソッドを持つ

- 内部保持データ
  - `row`  : 現在行（plain hashref）
  - `table`: `where` / `having` 実行時の全行テーブル（`grep_concat` の相対行取得に使用）
  - `idx`  : 現在行の `$table` 内インデックス（0始まり）

- 設計上の制約
  - ブレスされたオブジェクトとして返すため、`$tbl->grep_concat(...)` のようなメソッド呼び出しが可能
  - `$tbl->{col}` によるハッシュアクセスは `tie` 経由で `HashQuery::Value` を返す
  - `where` / `having` の両方で共通して使用する
  - 集約関数は持たない

### HashQuery::Value

- 役割
  - `$tbl->{col}` が返す値オブジェクト
  - カラム値を保持し、条件関数・変換関数を呼び出せる
  - 数値・文字列・真偽値コンテキストに応じたオーバーロードを持つ

- 仕様
  - 生の値そのものは返さない（オーバーロードで各コンテキストに適応する）
  - 未定義値も値オブジェクトとして扱い、各関数側で判定する
  - `like` / `not_like` / `between` / `in` / `not_in` / `asNull` などのメソッドを提供する

### HashQuery::WhereContext

- 現時点では空のパッケージとして存在する（ネームスペース予約）

### 10.1 値参照メソッド

- 呼び出し方法

```perl
$tbl->{col}
$_->{col}
```

- 入力
  - `col` : カラム名（文字列）

- 戻り値
  - `HashQuery::Value` のインスタンス

- 仕様
  - 指定カラムに対応する値オブジェクトを返す
  - 生の値そのものは返さない
  - 値オブジェクトは、条件関数や変換関数を呼び出せる
  - 値オブジェクトは、比較や真偽判定で利用できるように振る舞う
  - 未定義値も値オブジェクトとして扱い、各関数側で判定する

- 使用例

```perl
query $table,
    as $tbl,
    where { $tbl->{score} > 10 };
```

### 10.2 like

- 用途
  - 値に対するパターン一致判定

- 呼び出し方法

```perl
$tbl->{col}->like($pattern)
```

- 入力
  - `$pattern` : パターン文字列（`%` は任意文字列、`_` は任意1文字）

- 戻り値
  - 真偽値（一致すれば 1、しなければ 0）
  - 対象値が `undef` の場合は 0 を返す

- 仕様
  - `%` は任意文字列、`_` は任意1文字として扱う
  - 現在レコードの指定カラム値を対象に評価する

- 使用例

```perl
query $table,
    as $tbl,
    where { $tbl->{name}->like('ab%') };
```

### 10.3 not_like

- 用途
  - `like` の否定条件

- 呼び出し方法

```perl
$tbl->{col}->not_like($pattern)
```

- 入力
  - `$pattern` : パターン文字列（`%` は任意文字列、`_` は任意1文字）

- 戻り値
  - 真偽値（不一致であれば 1、一致すれば 0）
  - 対象値が `undef` の場合、`like` が 0 を返すため `not_like` は 1 を返す

- 仕様
  - `like` と同じ規則で評価した結果を反転して返す

- 使用例

```perl
query $table,
    as $tbl,
    where { $tbl->{name}->not_like('%error%') };
```

### 10.4 between

- 用途
  - 値が範囲内にあるかの判定

- 呼び出し方法

```perl
$tbl->{col}->between($min, $max)
```

- 入力
  - `$min` : 下限値（数値または文字列スカラー。末尾に `!` を付けると排他境界）
  - `$max` : 上限値（数値または文字列スカラー。末尾に `!` を付けると排他境界）

- 戻り値
  - 真偽値（範囲内であれば 1、範囲外であれば 0）
  - 対象値が `undef` の場合は 0 を返す

- 仕様
  - `!` なし：指定カラム値が下限以上かつ上限以下であるかを判定する（境界値を含む）
  - `!` あり：該当境界値を含まない（排他）として判定する
  - `!` は演算子ではなく、文字列末尾に付加する記法として扱う

- 使用例

```perl
# 10以上20以下（境界値を含む）
query $table,
    as $tbl,
    where { $tbl->{score}->between(10, 20) };

# 10以上20未満（上限を排他）
query $table,
    as $tbl,
    where { $tbl->{score}->between(10, '20!') };
```

### 10.5 in

- 用途
  - 値が候補集合に含まれるかの判定

- 呼び出し方法

```perl
$tbl->{col}->in([ $v1, $v2, ... ])
$tbl->{col}->in($v1, $v2, ...)
```

- 入力
  - 配列リファレンス（候補値のリスト）、またはフラットなリスト渡し

- 戻り値
  - 真偽値（候補集合に含まれれば 1、含まれなければ 0）
  - 対象値が `undef` の場合は 0 を返す

- 仕様
  - 指定カラム値が候補集合のいずれかと一致した場合に真とする
  - 引数が配列リファレンス 1つの場合はその中身を候補集合とする
  - 引数が複数のスカラーの場合はそれらを候補集合として扱う

- 使用例

```perl
query $table,
    as $tbl,
    where { $tbl->{kind}->in([ 'x', 'y' ]) };
```

### 10.6 not_in

- 用途
  - `in` の否定条件

- 呼び出し方法

```perl
$tbl->{col}->not_in([ $v1, $v2, ... ])
$tbl->{col}->not_in($v1, $v2, ...)
```

- 入力
  - 配列リファレンス（候補値のリスト）、またはフラットなリスト渡し

- 戻り値
  - 真偽値（候補集合に含まれなければ 1、含まれれば 0）

- 仕様
  - `in` と同じ引数形式を受け付ける
  - `in` の評価結果を反転して返す

- 使用例

```perl
query $table,
    as $tbl,
    where { $tbl->{kind}->not_in([ 'x', 'y' ]) };
```

### 10.7 asNull

- 用途
  - Null 相当値を別値へ置き換えて扱う

- 呼び出し方法

```perl
$tbl->{col}->asNull($default)
```

- 入力
  - `$default` : 置き換え後のデフォルト値（スカラー値）

- 戻り値
  - 対象値が `undef` または空文字の場合は `$default`
  - 対象値が存在する場合は対象値そのもの（スカラー）

- 仕様
  - 真偽値関数ではなく、変換・加工関数として扱う

- 使用例

```perl
# Null のレコードを 'N/A' として扱い、絞り込みに利用する
query $table,
    as $tbl,
    where { $tbl->{name}->asNull('N/A') ne 'N/A' };
```

### 10.8 grep_concat

- 用途
  - 現在レコードが条件に一致した場合に、前後行を含むコンテキストを連結して返す

- 呼び出し方法

```perl
$tbl->grep_concat($condition, $start, $end)
```

- 入力
  - `$condition` : 検索条件（文字列スカラー。現在レコードの全フィールド値に対する部分一致で評価する）
  - `$start` : 取得開始位置（整数。`0` は現在行、`-1` は1行前、`1` は1行後。省略時は 0）
  - `$end` : 取得終了位置（整数。省略時は `$start` と同値）

- 戻り値
  - 条件に一致した場合：指定範囲の行を連結した文字列
  - 条件に一致しない場合：空文字

- 仕様
  - 判定主体は現在レコードとする
  - 現在レコードが `$condition` に一致した場合のみ、指定した相対範囲の行を取得して連結する
  - 本関数は真偽値関数ではなく、文字列を返すレコード単位コンテキスト関数として扱う
  - 集約関数とは別物として扱う

- 使用例

```perl
# 'error' を含む行とその前後1行ずつを連結して取得する
query $table,
    as $tbl,
    where { $tbl->grep_concat('error', -1, 1) ne '' };
```

## 11. having

```perl
having {
    count_by(qw/a b/) > 1
};
```

- 戻り値

```perl
{
  having => CODE
}
```

- 引数
  - コードブロック（真偽値を返す条件式）

- 仕様
  - `having` が返すのは、`having` というキーを持つ深さ1のハッシュリファレンスである
  - `where` 後の Table を対象に評価する
  - 行単位で評価する
  - 条件成立行のみ残す
  - `having` 内で使う集約関数は、条件式で評価可能なスカラー値を返す
  - `as` が指定されている場合は、`having` の中でも `$tbl->{a}` のように現在行を参照できる
  - `having` の実装でも、`as` で受けた変数参照に対して現在行の hash リファレンスを割り当てる形を前提とする
  - `having` における `$tbl` や `$_` も、plain な hash リファレンスではなく `HashQuery::RowHash` のインスタンスとして扱う
  - `having` でも `$tbl->{a}` や `$_->{a}` が返す値は `HashQuery::Value` のインスタンスであり、`like` / `between` / `in` などを呼べる値オブジェクトである
  - `having` 実行時の現在行、および集約関数が参照する Table は `HashQuery::HavingContext` に保持させる
  - `having` 実行時には、`HashQuery::HavingContext` に保持された現在行と同じ行を、`as` で受けた変数参照にも割り当てる
  - `as` を指定しない場合でも、`$_` には current row に対応した同等の view を割り当てる
  - `having` の実装は、集約関数と alias 変数で完結させる
  - `having` のために caller package や node package のような追跡機構を前提にしない

- 使用可
  - 集約関数
  - `count_by`, `max_by`, `min_by`, `first_by`, `last_by` は明示的に export して利用可能にする
  - `as` が指定されている場合の alias 変数

## 12. 集約仕様

```perl
count_by(qw/a b/)
```

- 現在行のa,b値をキーとして
- `where` 後の Table 全体から一致件数を取得する
- 集約関数は、現在行に対応する集約値をスカラーとして返す

### 12.1 HashQuery::HavingContext による実現

- 役割
  - `having` 実行時の現在行と集約対象 Table を保持する
  - 集約関数が参照する計算コンテキストを提供する

- 保持情報
  - 現在行
  - `where` 後の Table

- 実現方式
  - `having` 実行時に、各レコード評価のたびに現在行を `HashQuery::HavingContext` に設定する
  - 同時に、集約対象である `where` 後の Table 全体を `HashQuery::HavingContext` に設定する
  - `count_by` などの集約関数は、このコンテキストを参照して現在行に対応する集約値を返す
  - alias 変数や `$_` による値参照と、集約関数による集合参照を同じ評価式内で併用できるようにする

## 13. 集約対象

- 必ず `where` 後の Table
- `query` の入力直後を直接集約対象にはしない

## 14. 集約最適化

- `having` 前に一度だけ前計算
- 例：
  - a,bごとの件数mapを生成しキャッシュ

## 15. 集約関数

```perl
count_by(qw/a b/)
max_by('c', qw/a b/)
min_by('c', qw/a b/)
first_by(qw/a b/)
last_by(qw/a b/)
```

- **having内のみ使用可**
- 返り値の考え方
  - `count_by` は件数を返す
  - `max_by` は最大値を返す
  - `min_by` は最小値を返す
  - `first_by` は現在行に対応する先頭側の値を返すための評価値として使う
  - `last_by` は現在行に対応する末尾側の値を返すための評価値として使う

### 15.1 count_by

- 呼び出し方法

```perl
count_by($col1, $col2, ...)
```

- 入力
  - `$col1, $col2, ...` : グループキーとなる1個以上のカラム名（文字列リスト）

- 戻り値
  - 現在行と同じグループキーを持つレコード数（整数）

- 仕様
  - 現在行の指定カラム値をグループキーとして取得する
  - `where` 後の Table 全体から同じキーを持つレコード数を数える

- 使用例

```perl
# a, b の組み合わせが2件以上のレコードのみ残す
query $table,
    as $tbl,
    having { count_by(qw/a b/) > 1 };
```

### 15.2 max_by

- 呼び出し方法

```perl
max_by($target, $col1, $col2, ...)
```

- 入力
  - `$target` : 最大値を求める対象カラム名（スカラー文字列 1つ。第一引数）
  - `$col1, $col2, ...` : グループキーとなるカラム名（文字列リスト。第二引数以降）

- 戻り値
  - グループ内の `$target` カラムの最大値（数値）
  - グループ内に有効値が存在しない場合は `undef`

- 仕様
  - 現在行のグループキーに属するレコード集合を対象とする
  - 比較は数値演算子（`>`）で行うため、文字列型カラムには使用しないこと
  - `undef` 値のレコードは比較対象から除外する

- 使用例

```perl
# a, b グループ内の c の最大値が 100 より大きいレコードのみ残す
query $table,
    as $tbl,
    having { max_by('c', qw/a b/) > 100 };
```

### 15.3 min_by

- 呼び出し方法

```perl
min_by($target, $col1, $col2, ...)
```

- 入力
  - `$target` : 最小値を求める対象カラム名（スカラー文字列 1つ。第一引数）
  - `$col1, $col2, ...` : グループキーとなるカラム名（文字列リスト。第二引数以降）

- 戻り値
  - グループ内の `$target` カラムの最小値（数値）
  - グループ内に有効値が存在しない場合は `undef`

- 仕様
  - 現在行のグループキーに属するレコード集合を対象とする
  - 比較は数値演算子（`<`）で行うため、文字列型カラムには使用しないこと
  - `undef` 値のレコードは比較対象から除外する

- 使用例

```perl
# a, b グループ内の c の最小値が 0 以上のレコードのみ残す
query $table,
    as $tbl,
    having { min_by('c', qw/a b/) >= 0 };
```

### 15.4 first_by

- 呼び出し方法

```perl
first_by($col1, $col2, ...)
```

- 入力
  - `$col1, $col2, ...` : グループキーとなる1個以上のカラム名（文字列リスト）

- 戻り値
  - 現在行がグループ内の先頭行であれば 1、そうでなければ 0（真偽値）

- 仕様
  - 現在行と同じグループに属するレコード群のうち、現在行が入力 Table の行順における先頭要素かを判定する

- 使用例

```perl
# a, b グループごとに先頭行のみを残す
query $table,
    as $tbl,
    having { first_by(qw/a b/) };
```

### 15.5 last_by

- 呼び出し方法

```perl
last_by($col1, $col2, ...)
```

- 入力
  - `$col1, $col2, ...` : グループキーとなる1個以上のカラム名（文字列リスト）

- 戻り値
  - 現在行がグループ内の末尾行であれば 1、そうでなければ 0（真偽値）

- 仕様
  - 現在行と同じグループに属するレコード群のうち、現在行が入力 Table の行順における末尾要素かを判定する

- 使用例

```perl
# a, b グループごとに末尾行のみを残す
query $table,
    as $tbl,
    having { last_by(qw/a b/) };
```

## 16. フィルタ整理

- `where`  : 行フィルタ
- `having` : 集約フィルタ
- `select` : 列制御のみ

## 17. 設計制約

- `from`        : なし
- `group_by`    : なし
- `distinct`    : なし
- 集約          : `having`のみ
- 実行主体      : `query`のみ
- 外部実行層    : なし

## 18. ネームスペース

- 基本パッケージ名は `HashQuery` とする
- `where` / `having` で使う tie 変数や値オブジェクトは、`HashQuery` の内部パッケージとして定義する
- `having` の集約関数も、`HashQuery` の内部パッケージとして定義する
- `having` の集約関数は export で提供し、`where` / `having` の alias 変数は `as` で受けた変数参照に対して割り当てる
- caller package や node package のような追跡機構を実装前提にしない
- パッケージ構成は次のとおり
  - `HashQuery`
    - 実行主体本体
  - `HashQuery::RowHash`
    - `$tbl` / `$_` の実体。ブレスされたオブジェクトとして返し、`tie` によるハッシュアクセスと `grep_concat` メソッドを提供する。内部に現在行・テーブル全体・行インデックスを保持する
  - `HashQuery::Value`
    - カラム値を保持する値オブジェクト。条件関数・変換関数および数値・文字列・真偽値オーバーロードを持つ
  - `HashQuery::WhereContext`
    - 現時点では空のパッケージ（ネームスペース予約）
  - `HashQuery::HavingContext`
    - `having` 実行時の現在行と `where` 後の Table を保持し、集約関数に計算コンテキストを提供する

## 19. query責務

- 第一引数の Table 受け取り
- 第二引数以降のノード解釈
- `as`解釈
- `select` / `except` 解釈
- 出力カラム確定
- `_row` カラム付加（各行に 0 始まりの行番号を付与）
- `where`実行
- `having`前計算
- `having`実行
- `select`列射影（`_row` は出力から除外）
- AOH返却

## 20. フル例

```perl
my $tbl;
my $result = query
    $table,
    as $tbl,
    select {
        except => [ 'c' ]
    },
    where {
        $tbl->{a} gt 'abc'
        and $tbl->{b} >= 10
        and $tbl->{b} <= 20
    },
    having {
        $tbl->{c} > 100
        and
        count_by(qw/a b/) > 1
        and max_by('c', qw/a b/) > 100
    };

my $cnt = scalar @$result;
```
